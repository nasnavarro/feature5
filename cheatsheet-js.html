DOM: Selección y Nodos

El DOM (Document Object Model) permite a JavaScript acceder y manipular la estructura del documento. Podemos seleccionar elementos únicos o listas de nodos, y movernos a través de su jerarquía.


          // Selección
          const card = document.querySelector(".card");
          const items = document.querySelectorAll("li"); // Devuelve NodeList

          // Navegación entre nodos
          const padre = el.parentElement;
          const hijos = el.children;
          const siguiente = el.nextElementSibling;

Reglas rápidas:

  • Usa querySelector como estándar para flexibilidad total.
  • querySelectorAll devuelve una NodeList que puedes recorrer con forEach.
  • Diferencia entre children (solo elementos HTML) y childNodes (incluye texto y comentarios).

DOM: Manipulación y CSS

Permite crear nuevos elementos desde cero, cambiar su contenido de texto o HTML, y gestionar sus estilos mediante clases de CSS de forma eficiente.


          // Crear e insertar
          const li = document.createElement("li");
          li.textContent = "Nuevo Item";
          document.querySelector("ul").append(li);

          // Gestión de clases (Recomendado)
          el.classList.add("active");
          el.classList.toggle("hidden");

          // Eliminar
          el.remove();

Reglas rápidas:

  • Prefiere textContent sobre innerHTML por seguridad (evita XSS).
  • Usa classList en lugar de style.prop para mantener la separación de conceptos (estilos en CSS, lógica en JS).
  • append() permite añadir varios elementos o texto a la vez, a diferencia de appendChild().

DOM: Eventos y Delegación

Los eventos detectan la interacción del usuario. La delegación de eventos permite manejar múltiples elementos hijos escuchando únicamente al padre, lo cual es vital para listas dinámicas.


          // Evento básico
          btn.addEventListener("click", (e) => {
            console.log("Click en:", e.target);
          });

          // Delegación de eventos
          ul.addEventListener("click", (e) => {
            if (e.target.matches("button.remove")) {
              e.target.closest("li").remove();
            }
          });

Reglas rápidas:

  • Usa e.preventDefault() en formularios para evitar que la página se recargue al enviar.
  • e.target es el elemento real donde se hizo click; e.currentTarget es el elemento que tiene el listener.
  • La delegación ahorra memoria al no crear cientos de listeners individuales.

Destructuring (Objetos y Arrays)

Sintaxis que permite extraer valores de objetos o elementos de arrays y asignarlos a variables de forma directa y limpia.


          // Objetos
          const user = { id: 1, name: "Ana", role: "admin" };
          const { name, role: userRole, city = "Madrid" } = user;

          // Arrays
          const colors = ["red", "blue", "green"];
          const [first, second] = colors;
          const [,, third] = colors; // Salta posiciones

Reglas rápidas:

  • Puedes renombrar variables usando clave: nuevoNombre.
  • Puedes asignar valores por defecto si la propiedad no existe.
  • Es muy útil en los parámetros de las funciones: const fn = ({ name }) => ....

Spread y Rest Operator (...)

El operador de tres puntos sirve para expandir elementos (Spread) o para agruparlos (Rest), facilitando la inmutabilidad de los datos.


          // Spread: Copiar y extender
          const original = [1, 2];
          const copia = [...original, 3]; // [1-3]

          const user = { name: "Data" };
          const updated = { ...user, role: "admin" };

          // Rest: Capturar el resto
          const { role, ...publicData } = updated;
          const [first, ...others] = [4-6];

Reglas rápidas:

  • Spread crea una **copia superficial** (shallow copy); cuidado con objetos anidados.
  • Úsalo para fusionar objetos sin mutar los originales.
  • Rest siempre debe ser el último elemento en una desestructuración o lista de parámetros.

Bucles Avanzados: for...of / for...in

JavaScript ofrece formas específicas para iterar sobre valores de colecciones o sobre las propiedades de un objeto.


          // for...of: Para valores (Arrays, NodeLists)
          for (const item of ["a", "b"]) {
            console.log(item);
          }

          // for...in: Para claves (Objetos)
          const user = { name: "Ana", age: 25 };
          for (const key in user) {
            console.log(key, user[key]);
          }

Reglas rápidas:

  • for...of es para elementos iterables (lo que "hay dentro").
  • for...in es para propiedades de objetos (las "etiquetas").
  • Para objetos, a menudo es mejor usar Object.entries(obj).forEach(([key, value]) => ...).

LocalStorage y JSON

Permite guardar datos en el navegador que persisten incluso tras cerrar la pestaña. Como solo guarda strings, necesitamos JSON para manejar objetos y arrays.


          // Guardar objeto
          const tasks = [{ id: 1, text: "Aprender JS" }];
          localStorage.setItem("myTasks", JSON.stringify(tasks));

          // Leer objeto
          const raw = localStorage.getItem("myTasks");
          const myTasks = raw ? JSON.parse(raw) : [];

          // Borrar
          localStorage.removeItem("myTasks");
          localStorage.clear(); // Borra todo

Reglas rápidas:

  • JSON.stringify convierte datos a texto para guardar.
  • JSON.parse convierte el texto recuperado de vuelta a su tipo original.
  • El límite de almacenamiento suele ser de unos 5MB por dominio.
