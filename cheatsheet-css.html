Conceptos Flexbox: Ejes y Terminología

Flexbox es un modelo de diseño unidimensional que distribuye el espacio basándose en dos ejes: el eje principal (main axis) y el eje transversal (cross axis). La dirección del eje principal no es fija; depende de la propiedad flex-direction. El eje perpendicular al principal se conoce automáticamente como eje transversal.


          /* Concepto de ejes */
          .container {
            display: flex; /* Activa el contexto flex */
            flex-direction: row; /* Eje principal horizontal */
            /* Eje transversal es vertical */
          }

Reglas rápidas:

  • Main Start / Main End: Puntos donde inicia y termina el flujo de los elementos.
  • Cross Axis: Dirección perpendicular al flujo principal.
  • Flexbox es ideal para componentes y diseños a pequeña escala; para layouts grandes se recomienda Grid.

Contenedor Flex: Propiedades del Padre

Estas propiedades se aplican al elemento contenedor para controlar el comportamiento global de sus hijos. Permiten definir la dirección, si el contenido debe saltar a una nueva línea y cómo se distribuye el espacio sobrante.


          .flex-container {
            display: flex;
            flex-direction: row; /* row | column | row-reverse | column-reverse */
            flex-wrap: wrap; /* permite que los ítems salten de línea */
            justify-content: space-between; /* Alineación en eje principal */
            align-items: center; /* Alineación en eje transversal (una línea) */
            align-content: stretch; /* Alineación de líneas (solo con wrap) */
            gap: 15px; /* Espacio entre elementos */
          }

Reglas rápidas:

  • justify-content: Controla el eje principal.
  • align-items: Alinea ítems dentro de su línea actual en el eje transversal.
  • align-content: Solo funciona si existe flex-wrap: wrap y hay múltiples líneas de contenido.

Elementos Flex: Propiedades de los Hijos

Los elementos hijos (flex items) pueden alterar su tamaño inicial para llenar el espacio disponible o encogerse para evitar desbordamientos. También pueden romper la alineación general del padre de forma individual.


          .flex-item {
            flex-grow: 1; /* Proporción de crecimiento */
            flex-shrink: 0; /* Evita que el elemento se encoja */
            flex-basis: 200px; /* Tamaño inicial antes de distribuir espacio */
            align-self: flex-end; /* Sobrescribe align-items del padre */
            order: 2; /* Cambia el orden visual */
          }

          /* Recomendado: Shorthand */
          .flex-item-pro {
            flex: 1 1 200px; /* grow | shrink | basis */
          }

Reglas rápidas:

  • Usa siempre el shorthand flex en lugar de las propiedades individuales para que los valores se ajusten inteligentemente.
  • order cambia la posición visual, pero no el orden real en el HTML (cuidado con la accesibilidad).
  • flex-shrink: 0 es vital para mantener el tamaño original de un elemento si el contenedor es pequeño.

Modelo de Caja (Box Model)

Cada elemento HTML se considera una caja rectangular. El modelo de caja define cómo se calcula el espacio total de un elemento sumando sus partes: contenido, relleno interno, borde y margen externo.


          .card {
            box-sizing: border-box; /* Muy recomendado */
            width: 300px;
            padding: 16px; /* Espacio por dentro */
            border: 1px solid #ddd; /* El límite de la caja */
            margin: 20px; /* Espacio por fuera */
          }

Reglas rápidas:

  • content: El texto o imágenes reales.
  • padding: Crea "aire" entre el contenido y el borde.
  • margin: Separa la caja de otros elementos.
  • box-sizing: border-box: Asegura que el padding y el border no aumenten el ancho total definido para el elemento.

Posicionamiento: Static, Relative, Absolute, Fixed y Sticky

La propiedad position determina cómo se ubica un elemento en el documento. Permite sacarlo del flujo normal o fijarlo en una posición específica de la pantalla.


          .parent { position: relative; } /* Referencia para hijos */

          .badge {
            position: absolute;
            top: 10px;
            right: 10px;
          }

          .navbar {
            position: sticky;
            top: 0;
          }

Reglas rápidas:

  • relative: Mantiene el espacio original pero permite mover el elemento o servir de referencia.
  • absolute: Se posiciona respecto al ancestro más cercano con posición relativa.
  • fixed: Se queda pegado a la pantalla sin importar el scroll.
  • sticky: Se comporta como relativo hasta que alcanza un punto de scroll definido, donde se queda "pegado".

Selectores y Especificidad

Los selectores indican a qué elementos HTML aplicar estilos. La especificidad es la jerarquía que decide qué regla gana cuando hay conflictos. Cuanto más específico sea el selector, más "fuerza" tendrá.


          /* Jerarquía de fuerza (de menos a más) */
          p { color: red; } /* Por etiqueta (1) */
          .clase { color: blue; } /* Por clase (10) */
          #id { color: green; } /* Por ID (100) - Muy fuerte */

          /* Ejemplo combinado */
          #header .nav a:hover { color: orange; }

Reglas rápidas:

  • Evita usar IDs para estilos; las clases son más reutilizables y fáciles de mantener.
  • El modificador !important rompe la cascada y debe evitarse en favor de una mejor especificidad.
  • Las pseudo-clases como :hover o :disabled seleccionan elementos basados en su estado actual.

Animaciones y Transiciones

Las transiciones permiten cambios suaves de un estado a otro en un tiempo determinado. Las animaciones con @keyframes permiten crear secuencias complejas de varios pasos sin necesidad de JavaScript.


          /* Transición simple */
          .btn {
            transition: transform 0.3s ease, background 0.2s;
          }
          .btn:hover { transform: scale(1.1); }

          /* Animación compleja */
          @keyframes slideIn {
            from { opacity: 0; transform: translateY(20px); }
            to { opacity: 1; transform: translateY(0); }
          }

          .modal {
            animation: slideIn 0.5s ease both;
          }

Reglas rápidas:

  • Usa transition para interacciones rápidas (hover, focus).
  • Usa animation-fill-mode: both para que el elemento mantenga el estado final de la animación.
  • Es mejor animar transform y opacity que propiedades de geometría (top, width) para mejorar el rendimiento (GPU).
