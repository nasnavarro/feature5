Cheatsheet - HTML

Guía para escribir HTML5 moderno de un modo semántico: estructura, texto, enlaces, formularios, listas, etc ... de un modo accesible y que sigan buenas prácticas de maquetación y diseño.

Estructura base (plantilla mínima)

Define el esqueleto fundamental de un documento HTML5.

              
                <!doctype html>
                <html lang="es">
                <head>
                  <meta charset="utf-8" />
                  <title>Mi página</title>
                </head>
                <body>
                  <h1>Hola HTML</h1>
                </body>
                </html>

Reglas rápidas:

  • <!doctype html> siempre arriba.
  • lang, meta charset y viewport casi siempre.
  • Títulos: 1 solo h1 principal por página (buena práctica).

Semántica (layout típico)

Etiquetas que describen la estructura y el significado del contenido.

              
                <header>
                  <nav>
                    <a href="/">Home</a>
                  </nav>
                </header>
                <main>
                  <article>
                    <h2>Título del post</h2>
                  </article>
                </main>

Cuándo usar:

  • header/footer: cabecera/pie de página o de una sección.
  • nav: bloque de navegación.
  • main: contenido principal (1 por página).
  • section: agrupación temática.
  • article: pieza independiente reutilizable.
  • aside: contenido secundario (relacionado).

Texto, enlaces y rutas

Organización de contenido textual e hipervínculos.

              
                <h1>Título principal</h1>
                <p>Texto con <strong>importancia</strong>.</p>
                <a href="https://ejemplo.com" target="_blank" rel="noopener noreferrer">
                  Enlace externo
                </a>

Reglas rápidas:

  • strong = importancia, em = énfasis (no solo "negrita" y "cursiva").
  • En enlaces con target="_blank" usa rel="noopener noreferrer".
  • alt en imágenes: describe el contenido o función.

Listas

Agrupación de elementos con o sin un orden específico.

              
                <ul>
                  <li>Elemento no ordenado</li>
                </ul>
                <ol>
                  <li>Paso 1</li>
                  <li>Paso 2</li>
                </ol>

Cuándo usar:

  • ul: lista sin orden.
  • ol: pasos / ranking / orden.

Formularios (lo esencial)

Interacción y recolección de datos del usuario.

              
                <form action="/enviar" method="post">
                  <label for="name">Nombre</label>
                  <input id="name" name="name" type="text" required />
                  <button type="submit">Enviar</button>
                </form>

Reglas rápidas:

  • label + for debe apuntar al id del campo.
  • Atributos útiles: required, min, max, minlength, maxlength, pattern.
  • name es clave: es lo que "viaja" al backend.

Multimedia (audio, vídeo e iframe)

Soporte nativo para reproducción y contenido externo.

              
                <audio controls src="audio/pista.mp3"></audio>
                <video controls width="480" src="video/demo.mp4"></video>
                <iframe src="https://www.youtube.com/embed/ID" title="Video"></iframe>

Reglas rápidas:

  • figure/figcaption para contenido con pie.
  • controls para permitir play/pausa.

Atributos clave (id, class, data-*)

Propiedades para identificación, estilos y datos personalizados.

              
                <section id="pricing" class="section--pricing">
                  <button class="btn" data-plan="pro" aria-label="Elegir plan Pro">
                    Elegir
                </button>
                </section>

Reglas rápidas:

  • id: único en la página (anclas, JS, accesibilidad).
  • class: reutilizable (CSS/JS).
  • data-*: datos personalizados para JS.
  • aria-label: ayuda a lectores de pantalla cuando el texto no es suficiente.
